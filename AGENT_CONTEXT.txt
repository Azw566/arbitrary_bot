## MISSION
Automated on-chain arbitrage bot targeting Ethereum mainnet (Sepolia testnet).
Monitors price discrepancies across Uniswap V2/V3, SushiSwap, and Curve for
two-way (A→B→A) and triangular (A→B→C→A) arbitrage using Aave V3 flash loans.

## DATA FLOW
1. BlockListener polls for new blocks every 300ms
2. Each new block triggers batch_get_pool_data() via Multicall3
   → Single RPC call fetches ~71 pools in ~0.9s
   → Decodes V2 reserves, V3 sqrtPriceX96, Curve dy
3. find_arbitrage_opportunities() compares prices per pair
   → Groups pools by pair → finds min/max price pools
   → net_profit = gross_spread - fees - gas_cost_pct
4. find_triangular_opportunities() builds directed graph
   → Detects A→B→C→A cycles where product(rates) > 1
5. Profitable opportunities queued in asyncio.PriorityQueue
6. Dispatch loop pulls from queue, drops stale (>10s old)
7. Executor.try_execute() acquires token locks → runs trade
   → dry_run=true: logs only | dry_run=false: sends tx

## KNOWN BOTTLENECKS TO OPTIMIZE
1. Gas + ETH price fetched from RPC EVERY block — should cache per-block
2. BlockListener uses 300ms polling — should use WebSocket subscription
3. Token info cache not persisted across restarts — use SQLite
4. Database writes happen synchronously — should use async queue
5. Subgraph queries run sequentially — should parallelize
6. Slippage is hardcoded — should model dynamically from pool depth
7. No MEV protection — front-running risk on all trades
8. No circuit breaker — bad RPC doesn't trigger active failover

## CONTRACTS
MULTIDEX.sol implements IFlashLoanReceiverV3 (Aave):
  1. executeOperation() called by Aave after flash loan
  2. Executes swaps through UniV2/UniV3/Curve routers
  3. Repays flash loan + 0.09% fee
  4. Profit stays in contract (withdrawable by owner)
  Owner functions: withdraw(), setOwner()
